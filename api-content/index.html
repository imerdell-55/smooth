即，makefile的作用就是使编译自动化，使我们可以通过一个简单的make命令来完成整个工程的编译，说到make，它是对makefile中指令进行解释的命令工具，大部分IDE都包含这个命令，这里仅指代Linux下GNU的 make，默认编译器是Unix下的GCC。 程序编译过程 对于c++的程序编译，需要经过编译和链接两步， * 编译(compile)：即先把原文件编译成中间代码文件，Unix下的.o文件， Win下的.obj文件 * 对于编译过程，需要确保语法正确，函数与变量的声明的正确。 * 链接(link)：把中间文件合成执行文件 * 对于链接过程，主要是链接函数和全局变量。对于有些中间件过多的项目，会将中间件打包，生成库文件杂，在Win下为.lib文件，在Unix下为.a/.so文件。 Makefile介绍 基本规则 如果该工程没有编译过，那么所有的C文件都要编译并链接 如果这个工程的某几个C文件被修改，则只编译被修改的C文件，并链接目标程序 如果头文件被修改，只用编译引用了这几个头文件的C文件，并链接 Makefile规则 target ... : prerequisites ... command ... ... 示例 edit : main.o kbd.o command.o display.o
